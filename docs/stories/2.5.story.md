## Story

**Como** usuário do portal Lizard Web,
**Quero** que a API sirva as imagens processadas dos couros de forma segura,
**Para que** o frontend possa exibir a imagem do couro na tela de Detalhe sem expor o sistema de arquivos da esteira.

---

## Acceptance Criteria

- [ ] AC-01: `GET /images/{encoded_path}` retorna o arquivo de imagem com `Content-Type: image/jpeg` quando o arquivo existir
- [ ] AC-02: `encoded_path` é o path relativo ao `RESULTS_DIR` configurado, URL-encoded
- [ ] AC-03: A API retorna `404` com `{"detail": "Imagem não disponível"}` se o arquivo não existir no disco
- [ ] AC-04: A API retorna `400` com `{"detail": "Path inválido"}` se path traversal for detectado (path resolvido fora de `RESULTS_DIR`)
- [ ] AC-05: A validação de segurança usa `os.path.realpath()` para resolver symlinks e `..` antes de comparar com `RESULTS_DIR`
- [ ] AC-06: O endpoint aceita apenas arquivos com extensão `.jpg` ou `.jpeg` (case-insensitive) — retorna `400` para outras extensões
- [ ] AC-07: `RESULTS_DIR` é lido de `config.py` — nunca hardcoded no router
- [ ] AC-08: `lizard-web/api/routers/images.py` não importa nenhum módulo de `lizard/`
- [ ] AC-09: O response usa `FileResponse` do FastAPI com `media_type="image/jpeg"`
- [ ] AC-10: Nenhum path absoluto ou fragmento do sistema de arquivos da esteira é exposto no response de erro

## Dev Notes

### Contrato de Endpoint
[Source: docs/architecture/system-architecture.md#4.6-imagens]

```
GET /images/{encoded_path}
```

- `encoded_path`: path relativo ao `RESULTS_DIR`, URL-encoded pelo frontend
- Retorna o arquivo de imagem diretamente (binário JPEG)
- `404` se arquivo não encontrado
- `400` se path traversal detectado

### Implementação de Segurança
[Source: docs/architecture/system-architecture.md#6-segurança]

A regra crítica: `os.path.realpath(path_resolvido)` deve **começar com** `RESULTS_DIR` resolvido.

```python
import os
from urllib.parse import unquote
from fastapi import APIRouter, HTTPException
from fastapi.responses import FileResponse
from api.config import RESULTS_DIR

router = APIRouter()

@router.get("/images/{encoded_path:path}")
def serve_image(encoded_path: str):
    # 1. Decodificar o path URL-encoded
    relative_path = unquote(encoded_path)

    # 2. Validar extensão antes de qualquer operação de filesystem
    ext = os.path.splitext(relative_path)[1].lower()
    if ext not in (".jpg", ".jpeg"):
        raise HTTPException(status_code=400, detail="Path inválido")

    # 3. Resolver o path completo e verificar confinamento
    results_dir_real = os.path.realpath(RESULTS_DIR)
    full_path = os.path.realpath(os.path.join(RESULTS_DIR, relative_path))

    if not full_path.startswith(results_dir_real + os.sep):
        raise HTTPException(status_code=400, detail="Path inválido")

    # 4. Verificar existência
    if not os.path.isfile(full_path):
        raise HTTPException(status_code=404, detail="Imagem não disponível")

    # 5. Servir o arquivo
    return FileResponse(full_path, media_type="image/jpeg")
```

> **Crítico — separador no `startswith`:** Usar `results_dir_real + os.sep` (não apenas `results_dir_real`) para evitar falso positivo onde `RESULTS_DIR=/results` e o path resolvido é `/results_malicious/file.jpg` — sem o `os.sep`, `startswith("/results")` retornaria `True` incorretamente.

> **Crítico — `{encoded_path:path}`:** O `:path` no path parameter do FastAPI permite que o valor contenha `/`, necessário para paths com subdiretórios (`results/2026-06-26/12345/image.jpg`).

### Como o Frontend Vai Usar Este Endpoint

O frontend constrói a URL da imagem a partir do campo `GrdResPat` + nome do arquivo (reconstruído pela Story 2.4). Exemplo de uso pelo frontend:

```typescript
// src/pages/HideDetail.tsx
const imageUrl = `/images/${encodeURIComponent(relativePath)}`;
// ou, se relativePath tem subpastas:
const imageUrl = `/images/${relativePath.split('/').map(encodeURIComponent).join('/')}`;
```

O proxy do Vite (`vite.config.ts`) encaminha `/images/*` para `http://localhost:8000/images/*` automaticamente.

### Segurança — Resumo das Proteções
[Source: docs/architecture/system-architecture.md#6-segurança]

| Ameaça | Proteção |
|--------|----------|
| Path traversal (`../../etc/passwd`) | `os.path.realpath()` + `startswith(RESULTS_DIR + os.sep)` |
| Symlink escape | `os.path.realpath()` resolve symlinks |
| Extensão arbitrária | Whitelist `.jpg`/`.jpeg` antes de tocar filesystem |
| Exposição de path interno | Mensagens de erro genéricas (`"Path inválido"`, `"Imagem não disponível"`) |
| Arquivos fora de `results/` | Comparação com `RESULTS_DIR` real |
