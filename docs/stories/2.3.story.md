# Story 2.3 — Endpoints GET /ops/{id} e GET /ops/{id}/hides

**Epic:** 2 — Backend API
**Story:** 2.3
**Status:** Done
**Criado por:** @sm (River)
**Data:** 2026-06-30

## Story

**Como** usuário do portal Lizard Web,
**Quero** acessar o resumo agregado de uma OP específica e a lista paginada de couros processados nela,
**Para que** o frontend possa exibir a tela de Detalhe da OP com métricas consolidadas e navegação pelos couros individualmente.

---

## Acceptance Criteria

- [x] AC-01: `GET /ops/{op_id}` retorna JSON com campos `op`, `total_hides`, `avg_area`, `avg_yield`, `class_distribution`
- [x] AC-02: `class_distribution` é lista de objetos `{ "class": str, "count": int, "pct": float }` ordenada por `count` decrescente
- [x] AC-03: `GET /ops/{op_id}` retorna 404 com `{"detail": "OP não encontrada"}` se `op_id` não existir no banco
- [x] AC-04: `GET /ops/{op_id}/hides` retorna JSON com campos `items`, `total`, `page`, `limit`
- [x] AC-05: Cada item de `items` contém `id`, `hide_num`, `processed_at`, `class`, `yield_pct`, `area`, `op`
- [x] AC-06: `GET /ops/{op_id}/hides` suporta query params `page` (default 1) e `limit` (default 50, max 200)
- [x] AC-07: `GET /ops/{op_id}/hides` retorna 404 se `op_id` não existir no banco
- [x] AC-08: Ordenação padrão dos couros: `hide_num` crescente (ordem de processamento)
- [x] AC-09: Toda query usa parameterized statements — sem concatenação de strings em SQL
- [x] AC-10: `lizard-web/api/routers/ops.py` não importa nenhum módulo de `lizard/`
- [x] AC-11: `avg_area` e `avg_yield` excluem registros NULL (consistente com PRD seção 4.2)
- [x] AC-12: `pct` em `class_distribution` é calculado como `round(count / total_hides * 100, 1)`

### Mapeamento SQLite → API

| Campo API | Coluna SQLite |
|-----------|--------------|
| `op` | `GrdOp` |

### Query Base — GET /ops/{op_id}

```python
# 1. Verificar existência da OP
cursor.execute(
    "SELECT COUNT(*) FROM grading_results WHERE GrdOp = ?",
    (op_id,)
)
if cursor.fetchone()[0] == 0:
    raise HTTPException(status_code=404, detail="OP não encontrada")

# 2. Métricas agregadas
cursor.execute("""
    SELECT
        GrdOp as op,
        COUNT(*) as total_hides,
        ROUND(AVG(GrdAre), 2) as avg_area,
        ROUND(AVG(GrdUsaPer), 2) as avg_yield
    FROM grading_results
    WHERE GrdOp = ?
""", (op_id,))
row = cursor.fetchone()

# 3. Distribuição de classes
cursor.execute("""
    SELECT
        GrdCla as class,
        COUNT(*) as count
    FROM grading_results
    WHERE GrdOp = ?
    GROUP BY GrdCla
    ORDER BY count DESC
""", (op_id,))
classes = cursor.fetchall()

# Calcular pct
total = row["total_hides"]
class_distribution = [
    {"class": c["class"], "count": c["count"], "pct": round(c["count"] / total * 100, 1)}
    for c in classes
]
```

### Query Base — GET /ops/{op_id}/hides

```python
# 1. Verificar existência da OP (mesmo padrão acima)

# 2. Total para paginação
cursor.execute(
    "SELECT COUNT(*) FROM grading_results WHERE GrdOp = ?",
    (op_id,)
)
total = cursor.fetchone()[0]

# 3. Página de couros
offset = (page - 1) * limit
cursor.execute("""
    SELECT
        rowid as id,
        GrdHidNum as hide_num,
        GrdDat as processed_at,
        GrdCla as class,
        GrdUsaPer as yield_pct,
        GrdAre as area,
        GrdOp as op
    FROM grading_results
    WHERE GrdOp = ?
    ORDER BY GrdHidNum ASC
    LIMIT ? OFFSET ?
""", (op_id, limit, offset))
items = cursor.fetchall()
```

> **Nota:** Confirmar nome da coluna PK no schema real. Se não existir `rowid` explícito, usar `rowid` implícito do SQLite — funciona mesmo sem declaração explícita. Verificar com `PRAGMA table_info(grading_results)`.

class HideItem(BaseModel):
