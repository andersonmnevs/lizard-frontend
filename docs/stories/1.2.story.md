### Conexão SQLite Read-Only
[Source: docs/architecture/system-architecture.md#6-segurança]

A conexão deve ser aberta em modo read-only para garantir que a API nunca modifique os dados do pipeline:

```python
# db.py
import sqlite3
from .config import SQLITE_PATH

def get_db_connection():
    conn = sqlite3.connect(f"file:{SQLITE_PATH}?mode=ro", uri=True)
    conn.row_factory = sqlite3.Row
    return conn
```

`sqlite3.Row` permite acesso por nome de coluna (`row["GrdOp"]`).

### Mapeamento de Colunas SQLite

### Modelos Pydantic Base
Criar estruturas mínimas para uso pelos routers do Epic 2:

```python
# models/op.py
from pydantic import BaseModel
from typing import Optional

class OpSummary(BaseModel):
    op: str
    last_updated: Optional[str]
    total_hides: int
    avg_yield: Optional[float]
    predominant_class: Optional[str]
```

### Segurança — Parameterized Queries
[Source: docs/architecture/system-architecture.md#6-segurança]

NUNCA concatenar strings em SQL. Usar sempre `?` como placeholder:

```python
# CORRETO
cursor.execute("SELECT * FROM grading_results WHERE GrdOp = ?", (op_id,))

# PROIBIDO
cursor.execute(f"SELECT * FROM grading_results WHERE GrdOp = '{op_id}'")
```
