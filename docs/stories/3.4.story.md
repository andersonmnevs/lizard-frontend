## Story

**Quero** visualizar os dados completos de um couro específico junto com a imagem processada com sobreposição de defeitos e navegar para o couro anterior ou próximo dentro da mesma OP,

### Reconstrução da URL da Imagem

A URL da imagem para o `<img src>` deve apontar para o endpoint `GET /images/{path}`. O path relativo precisa ser reconstruído com os mesmos campos usados pela API (Story 2.4):

```typescript
function buildImageUrl(hide: HideDetail): string {
  // O backend expõe image_available mas não o path completo.
  // O frontend solicita via endpoint /images/ usando o padrão de nomenclatura.
  // O path relativo segue: {date}-{time}-{op}-{hide_num}-{area}.jpg
  // Como o frontend não tem GrdResPat, usar endpoint direto via hide_id.
  //
  // ALTERNATIVA SIMPLES: Adicionar campo image_path ao response da API (Story 2.4)
  // se a reconstrução no frontend for problemática.
  //
  // Por ora, usar /images/hide/{hide_id} se a API expuser esse atalho,
  // ou solicitar ao @dev da Story 2.4 incluir campo image_url no response.
  return `/images/hide/${hide.id}`;
}
```

> **Decisão de design necessária:** O response de `GET /hides/{id}` não inclui o path completo da imagem (apenas `image_available: bool`). Duas opções:
> 1. **Adicionar `image_path` ao response da API** (modificação pequena na Story 2.4 ou via fix) — recomendado
> 2. **Adicionar endpoint auxiliar `GET /images/hide/{hide_id}`** que redireciona para o arquivo correto — mais simples para o frontend
>
> O @dev deve alinhar com o contrato da API antes de implementar esta story. Se necessário, abrir fix na Story 2.4.

### Componente ImageViewer

```typescript
// src/components/ui/ImageViewer.tsx
import { useState } from 'react';

interface Props {
  src: string;
  alt: string;
}

export default function ImageViewer({ src, alt }: Props) {
  const [zoomed, setZoomed] = useState(false);

  return (
    <div
      onClick={() => setZoomed(z => !z)}
      style={{ cursor: 'zoom-in', overflow: 'auto' }}
    >
      <img
        src={src}
        alt={alt}
        style={{
          maxWidth: zoomed ? 'none' : '100%',
          transform: zoomed ? 'scale(1.5)' : 'scale(1)',
          transformOrigin: 'top left',
          transition: 'transform 0.2s',
        }}
        onError={(e) => {
          // Fallback se a imagem falhar após image_available=true
          (e.target as HTMLImageElement).style.display = 'none';
        }}
      />
    </div>
  );
}
```

### Lógica do Componente HideDetail

```typescript
// src/pages/HideDetail.tsx
import { useEffect, useState } from 'react';
import { useParams, useNavigate, Link } from 'react-router-dom';
import client from '../api/client';
import { HideDetail as HideDetailType } from '../types/hide';
import ImageViewer from '../components/ui/ImageViewer';

export default function HideDetail() {
  const { opId, hideId } = useParams<{ opId: string; hideId: string }>();
  const navigate = useNavigate();
  const [hide, setHide] = useState<HideDetailType | null>(null);
  const [loading, setLoading] = useState(true);
  const [notFound, setNotFound] = useState(false);

  useEffect(() => {
    setLoading(true);
    setHide(null);
    client.get<HideDetailType>(`/hides/${hideId}`)
      .then(res => setHide(res.data))
      .catch(err => {
        if (err.response?.status === 404) setNotFound(true);
      })
      .finally(() => setLoading(false));
  }, [hideId]);

  const navigateToHide = (id: number) => navigate(`/ops/${opId}/hides/${id}`);

  if (loading) return <div>Carregando...</div>;
  if (notFound) return (
    <div>Couro não encontrado. <Link to={`/ops/${opId}`}>Voltar para a OP</Link></div>
  );
  if (!hide) return null;

  return (
    <div>
      {/* Dados do couro */}
      {/* Imagem ou "Imagem não disponível" */}
      {hide.image_available
        ? <ImageViewer src={`/images/hide/${hide.id}`} alt={`Couro ${hide.hide_num}`} />
        : <p>Imagem não disponível</p>
      }
      {/* Navegação */}
      <button
        disabled={!hide.prev_hide_id}
        onClick={() => hide.prev_hide_id && navigateToHide(hide.prev_hide_id)}
      >
        Couro anterior
      </button>
      <button
        disabled={!hide.next_hide_id}
        onClick={() => hide.next_hide_id && navigateToHide(hide.next_hide_id)}
      >
        Próximo couro
      </button>
      <Link to={`/ops/${opId}`}>Voltar para a OP</Link>
    </div>
  );
}
```

> **Nota:** `useParams` retorna `opId` do path `/ops/:opId/hides/:hideId`. Verificar que a rota está configurada exatamente com esses nomes em `App.tsx` (Story 1.3).
