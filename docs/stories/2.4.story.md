## Story

**Como** usuário do portal Lizard Web,
**Quero** acessar os detalhes completos de um couro específico incluindo indicadores de navegação para o couro anterior e próximo dentro da mesma OP,
**Para que** o frontend possa exibir a tela de Detalhe do Couro com todas as métricas e permitir navegação sequencial sem retornar à lista.

---

## Acceptance Criteria

- [ ] AC-01: `GET /hides/{hide_id}` retorna JSON com campos `id`, `hide_num`, `op`, `processed_at`, `class`, `yield_pct`, `area`, `image_available`, `prev_hide_id`, `next_hide_id`
- [ ] AC-02: `image_available` é `true` se o arquivo de imagem existir no disco, `false` caso contrário — sem erro
- [ ] AC-03: `prev_hide_id` é o `id` do couro com `hide_num` imediatamente anterior na mesma OP, `null` se for o primeiro
- [ ] AC-04: `next_hide_id` é o `id` do couro com `hide_num` imediatamente posterior na mesma OP, `null` se for o último
- [ ] AC-05: `GET /hides/{hide_id}` retorna 404 com `{"detail": "Couro não encontrado"}` se `hide_id` não existir
- [ ] AC-06: A reconstrução do path da imagem segue o padrão: `{GrdResPat}{GrdDat_date}-{GrdDat_time}-{GrdOp}-{GrdHidNum}-{GrdAre}.jpg`
- [ ] AC-07: A verificação de existência da imagem usa `os.path.isfile()` — nunca levanta exceção se o arquivo não existir
- [ ] AC-08: Toda query usa parameterized statements — sem concatenação de strings em SQL
- [ ] AC-09: `lizard-web/api/routers/hides.py` não importa nenhum módulo de `lizard/`
- [ ] AC-10: `yield_pct` e `area` podem ser `null` se o valor no banco for NULL

```python
# 1. Buscar couro pelo id
cursor.execute("""
    SELECT
        rowid as id,
        GrdHidNum as hide_num,
        GrdOp as op,
        GrdDat as processed_at,
        GrdCla as class,
        GrdUsaPer as yield_pct,
        GrdAre as area,

    WHERE rowid = ?
```

### Reconstrução do Path da Imagem
[Source: docs/architecture/system-architecture.md#4.3-localização-de-imagens]

O campo `GrdDat` armazena data e hora — extrair partes para montar o nome do arquivo:

```python
import os
from datetime import datetime

def build_image_path(result_path: str, processed_at: str, op: str, hide_num: str, area: float) -> str:
    """Reconstrói o path completo da imagem seguindo o padrão de nomenclatura."""
    # GrdDat formato esperado: "2026-06-26T14:32:00" ou "2026-06-26 14:32:00"
    dt = datetime.fromisoformat(processed_at.replace(" ", "T"))
    date_str = dt.strftime("%d.%m.%Y")   # ex: 26.06.2026
    time_str = dt.strftime("%H.%M.%S")   # ex: 14.32.00
    filename = f"{date_str}-{time_str}-{op}-{hide_num}-{area}.jpg"
    return os.path.join(result_path, filename)

def check_image_available(result_path: str, processed_at: str, op: str, hide_num: str, area) -> bool:
    """Verifica se o arquivo de imagem existe no disco. Nunca levanta exceção."""
    try:
        if not result_path or area is None:
            return False
        full_path = build_image_path(result_path, processed_at, op, hide_num, str(area))
        return os.path.isfile(full_path)
    except Exception:
        return False
```

> **Atenção:** O formato exato de `GrdDat` no banco (separador `T` ou espaço, presença de milissegundos) deve ser verificado na Story 2.3 ou via `PRAGMA`. Usar `.replace(" ", "T")` antes de `fromisoformat` para robustez.

> **Atenção:** O padrão de nomenclatura do arquivo (formato de data, separadores) deve ser confirmado verificando arquivos reais em `results/`. Se o formato diferir, ajustar `date_str`/`time_str` no `strftime`.

### Lógica de Navegação Prev/Next

Navegar por `hide_num` crescente dentro da mesma OP:

```python
# prev: couro com hide_num imediatamente anterior (menor hide_num mais próximo)
cursor.execute("""
    SELECT rowid as id
    FROM grading_results
    WHERE GrdOp = ? AND GrdHidNum < ?
    ORDER BY GrdHidNum DESC
    LIMIT 1
""", (row["op"], row["hide_num"]))
prev_row = cursor.fetchone()
prev_hide_id = prev_row["id"] if prev_row else None

# next: couro com hide_num imediatamente posterior (maior hide_num mais próximo)
cursor.execute("""
    SELECT rowid as id
    FROM grading_results
    WHERE GrdOp = ? AND GrdHidNum > ?
    ORDER BY GrdHidNum ASC
    LIMIT 1
""", (row["op"], row["hide_num"]))
next_row = cursor.fetchone()
next_hide_id = next_row["id"] if next_row else None
```

> **Nota:** `GrdHidNum` pode ser string ou numérico no banco. Se for string com zeros à esquerda (ex: `"001"`, `"002"`), a comparação `<` e `>` pode funcionar lexicograficamente. Verificar tipo real com `PRAGMA table_info`. Se necessário, usar `CAST(GrdHidNum AS INTEGER)` nas comparações.
