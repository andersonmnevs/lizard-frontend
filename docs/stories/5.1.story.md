## Dev Notes

### Arquivo systemd Unit File
[Source: docs/architecture/system-architecture.md#2-stack-tecnológica]
[Source: docs/architecture/system-architecture.md#10-decisões-arquiteturais — D-05, D-06]

```ini
# lizard-web/lizard-web.service
[Unit]
Description=Lizard Web — Portal de Consulta de Couros
After=network.target

[Service]
Type=simple
User=lizard
WorkingDirectory=/opt/lizard-web
Environment="SQLITE_PATH=/opt/lizard/data/results.db"
Environment="RESULTS_DIR=/opt/lizard/results"
ExecStart=/opt/lizard-web/api/.venv/bin/uvicorn api.main:app --host 0.0.0.0 --port 8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> **Nota:** Os paths (`/opt/lizard-web`, `/opt/lizard/data/results.db`, `User=lizard`) devem ser ajustados conforme a instalação real na esteira. O `DEPLOY.md` deve documentar como configurar esses valores.

### Servir Frontend Estático via FastAPI
[Source: docs/architecture/system-architecture.md#10-decisões-arquiteturais — D-06]

```python
# lizard-web/api/main.py — adicionar ao final, após todos os routers
import os
from pathlib import Path
from fastapi.staticfiles import StaticFiles

# Serve o frontend buildado
frontend_dist = Path(__file__).parent.parent / "frontend" / "dist"
if frontend_dist.exists():
    app.mount("/", StaticFiles(directory=str(frontend_dist), html=True), name="frontend")
```

> **Crítico — ordem das rotas:** O `StaticFiles` com `html=True` deve ser montado **após** todos os routers da API. Se montado antes, interceptará as rotas da API. O `html=True` faz o `index.html` ser servido para rotas não encontradas (necessário para SPA com react-router).

> **Crítico — `vite.config.ts`:** O proxy para `/api` não será necessário em produção (API e frontend no mesmo servidor). Verificar se o `client.ts` usa URL relativa (sem `http://localhost:8000`) — se usar URL absoluta hardcoded, ajustar para usar variável de ambiente `VITE_API_URL` ou deixar relativa (`/ops`, `/dashboard`, etc.).

### Verificação do client.ts para Produção

Em `frontend/src/api/client.ts`, verificar que a `baseURL` funciona em produção:

```typescript
// Correto para produção (API e frontend no mesmo servidor):
const client = axios.create({
  baseURL: import.meta.env.VITE_API_URL || '',  // vazio = URL relativa
});
```

Se `baseURL` estiver hardcoded como `http://localhost:8000`, o frontend não funcionará em outras máquinas da rede.
